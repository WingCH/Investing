# TradingView Forex Screener 自動導出工具 - 基本原理解說

## 概覽 Overview

- **工具名稱 Tool Name**: TradingView Forex Screener Auto Export System
- **策略類型 Strategy Type**: 數據抓取與監控工具 (Data Scraping & Monitoring)
- **目標市場 Target Market**: 外匯市場 (Forex)

## 核心原理 Core Thesis

### 問題定義
TradingView 的 Forex Screener 提供了強大的技術分析評級功能，能夠基於多個技術指標（移動平均線、震盪指標等）為每個貨幣對生成綜合評級（Strong Buy, Buy, Neutral, Sell, Strong Sell）。然而，免費用戶無法導出這些數據，也無法追蹤歷史評級變化，導致：

1. **數據無法留存**: 無法建立歷史記錄進行回測分析
2. **變化難以追蹤**: 需要手動記錄評級變化，耗時且容易遺漏
3. **即時性不足**: 無法在評級變化時立即收到通知

### 解決方案
本工具通過以下方式解決上述問題：

1. **自動化數據抓取**: 使用 Firecrawl AI 驅動的網頁抓取技術，定期獲取 TradingView Forex Screener 的最新數據
2. **結構化存儲**: 將數據存入 Google Sheets，建立三層數據結構（最新、歷史、變化日誌）
3. **智能比對**: 自動比對新舊數據，識別技術評級的變化
4. **即時通知**: 當評級發生變化時，透過 Bark 推送通知到手機

### 技術評級的意義
TradingView 的技術評級基於以下指標的綜合分析：
- **移動平均線**: EMA(10), SMA(20), SMA(50), SMA(100), SMA(200)
- **震盪指標**: RSI, Stochastic, CCI, ADX, MACD
- **其他指標**: Williams %R, Bull/Bear Power, Ultimate Oscillator

當多個指標同時發出一致信號時，技術評級會相應調整。評級變化通常意味著：
- **轉為 Buy/Strong Buy**: 多頭信號增強，可能是買入機會
- **轉為 Sell/Strong Sell**: 空頭信號增強，可能是賣出機會
- **轉為 Neutral**: 市場處於盤整或信號分歧狀態

### 工作流程
```
TradingView 網站
       ↓
Firecrawl 抓取 (每2小時)
       ↓
數據解析與格式化
       ↓
與上次數據比對
       ↓
    分支處理
   ↙      ↘
無變化    有變化
  ↓        ↓
存檔    發送通知 + 記錄變化
```

### 應用場景
1. **趨勢跟蹤**: 追蹤主要貨幣對的技術評級趨勢，發現潛在交易機會
2. **信號確認**: 結合自身交易策略，使用評級變化作為輔助確認信號
3. **回測研究**: 利用歷史數據研究技術評級與實際價格走勢的相關性
4. **多資產監控**: 同時監控數十個貨幣對，不遺漏任何重要信號

### 局限性
- **信號延遲**: 抓取頻率受限於 API 配額（建議每2小時），無法實現秒級監控
- **評級滯後**: 技術指標本身具有滯後性，評級變化通常在趨勢已形成後才出現
- **非交易建議**: 本工具僅提供數據監控，不構成交易建議，需結合其他分析方法
- **依賴第三方**: 依賴 TradingView 網站結構和 Firecrawl 服務的穩定性

## 成本效益 Cost-Benefit

### 成本
- **Firecrawl API**: 使用免費方案（每月 500 次請求），執行頻率為每 2 小時，每月約 360 次請求
- **Google Sheets**: 完全免費
- **Bark 通知**: 完全免費
- **N8N 部署**: 自架版本免費（僅需伺服器或本機運行）

**總成本: $0/月**

### 效益
- **時間節省**: 替代手動記錄，每天節省約 10-15 分鐘
- **不遺漏信號**: 24/5 自動監控，不會因人為疏忽錯過重要評級變化
- **數據積累**: 建立完整歷史數據庫，支援後續回測和分析
- **即時性**: 評級變化後 1-2 小時內即可收到通知

## 風險管理 Risk Management

1. **技術風險**
   - TradingView 網站結構變化可能導致抓取失敗 → 定期檢查 workflow 執行狀態
   - API 服務中斷 → 設定錯誤通知，及時發現問題

2. **數據風險**
   - 抓取數據可能不準確 → 定期與 TradingView 網站人工比對驗證
   - 歷史數據可能丟失 → 定期備份 Google Sheets 數據

3. **使用風險**
   - 過度依賴技術評級 → 僅作為輔助工具，不應作為唯一交易依據
   - 評級誤導 → 理解技術指標的局限性，結合基本面和市場情緒分析

## 改進方向 Future Improvements

1. **多來源驗證**: 整合其他 Screener（如 Investing.com）交叉驗證
2. **信號過濾**: 基於歷史準確率，僅通知高置信度的評級變化
3. **自動回測**: 自動分析評級變化後 N 天的實際價格表現
4. **AI 分析**: 結合 LLM 分析評級變化與新聞、經濟數據的關聯
5. **交易整合**: 與交易系統對接，實現半自動化交易決策輔助
